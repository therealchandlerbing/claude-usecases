## What Each Example Will Include

**1. Complete HTML File**
- Production-ready single-page portfolio
- Fully commented code explaining design decisions
- Example of sector-specific adaptations

**2. README Documentation**
- Client context and goals
- Design rationale (color, typography, layout choices)
- Content strategy decisions
- Vianeo data translation approach
- Outcome and impact

**3. Before/After Comparison**
- Raw Vianeo data input
- Translated narrative output
- Design evolution screenshots

**4. Lessons Learned**
- What worked well
- What to avoid
- Sector-specific insights
- Reusable patterns identified

## How to Use Examples

**For Learning:**
1. Read README to understand context
2. View live portfolio (or open HTML in browser)
3. Inspect code to see implementation
4. Compare Vianeo input to narrative output

**For Building:**
1. Find example matching your sector
2. Study design patterns used
3. Adapt color palette and imagery
4. Reference content structure
5. Customize for your client

**For Contributing:**
1. Build a portfolio using this skill
2. Document design decisions
3. Create example structure (HTML + README + screenshots)
4. Submit as example for others to learn from

---

## Contributing Examples

**Want to add an example?**

1. Build a complete portfolio using this skill
2. Get client approval to share (anonymize if needed)
3. Create directory: `examples/[sector-name]/`
4. Include:
   - `index.html` - Complete portfolio HTML
   - `README.md` - Documentation of decisions
   - `vianeo-input.md` - Raw data input (anonymized)
   - `screenshots/` - Visual documentation
5. Update this README with new example
6. Submit pull request or share with 360 team
