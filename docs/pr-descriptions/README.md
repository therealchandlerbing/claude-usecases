# Pull Request Descriptions

This directory contains pull request templates and descriptions for major features and implementations.

## Contents

All PR description files follow the naming convention `PR_DESCRIPTION_*.md` and provide:
- Comprehensive feature overviews
- Implementation details and technical decisions
- Testing and verification instructions
- Related documentation and resources

## Using PR Descriptions

When creating a pull request for a major feature:

1. Reference the appropriate PR description file for your feature
2. Follow the structure and level of detail provided
3. Update the description with any feature-specific details
4. Include all relevant testing and verification steps

## Available PR Descriptions

- **Contract Redlining Tool** - Automated contract review and redlining system
- **Cross-Linking Architecture** - Integration and data flow between systems
- **Financial Modeling** - Investment analysis and portfolio intelligence
- **Impact Templates** - Executive impact report generation system
- **Intelligence Extractor** - Partnership and funding intelligence extraction
- **Open Deep Research** - Multi-agent comprehensive research system
- **Sales Automator** - Intelligent sales automation and pipeline tracking
- **Testing Infrastructure** - Python and TypeScript testing setup

## Instructions Files

- `CREATE_PR_*.md` - Step-by-step instructions for creating PRs
- `PULL_REQUEST.md` - General pull request guidelines
